# state! Macro

The `state!` macro declares your application's TEE state with automatic sealing, key management, and encryption support.

## Syntax

```rust
state! {
    #[mrenclave]
    Type1,          // Reset on redeploy
    Type2,

    #[mrsigner]
    Type3,          // Persist across redeploys

    #[external]
    Type4,          // Generate encrypt/decrypt methods
}
```

Each section is optional. You can use any combination of tiers.

## Requirements for state types

Types used in `#[mrenclave]` and `#[mrsigner]` must implement:
- `Serialize` + `Deserialize` (for sealed storage)
- `Default` (for initial creation)
- `Clone` + `Debug`

Types used in `#[external]` must additionally derive `Encrypted`.

## Generated code

### `EnclaveState` (from `#[mrenclave]`)

```rust
pub struct EnclaveState {
    signing_key: ed25519_dalek::SigningKey,  // Auto-managed
    pub type1: Type1,
    pub type2: Type2,
}

impl EnclaveState {
    pub fn type1(&self) -> &Type1 { &self.type1 }
    pub fn type2(&self) -> &Type2 { &self.type2 }
}
```

### `SignerState` (from `#[mrsigner]`)

```rust
pub struct SignerState {
    master_key: [u8; 32],                   // Auto-managed
    pub current_key_version: u32,           // Starts at 1
    pub rotation_interval_days: u64,        // Default 90
    pub last_rotation: String,              // RFC3339
    pub next_rotation: String,              // RFC3339
    pub retired_keys: Vec<RetiredKeyEntry>, // For backward decryption
    pub type3: Type3,
}

impl SignerState {
    pub fn type3(&self) -> &Type3 { &self.type3 }
}
```

### `TeeState` (unified container)

```rust
pub struct TeeState {
    enclave: EnclaveState,  // Present if #[mrenclave] used
    signer: SignerState,    // Present if #[mrsigner] used
}
```

#### Always available

```rust
impl TeeState {
    pub fn initialize(seal_dir: &Path) -> Result<Self, SdkError>;
    pub fn seal(&self, seal_dir: &Path) -> Result<(), SdkError>;
}
```

#### With `#[mrenclave]`

```rust
impl TeeState {
    pub fn enclave(&self) -> &EnclaveState;
    pub fn enclave_mut(&mut self) -> &mut EnclaveState;
    pub fn sign_response(&self, body: &[u8], request_id: &str) -> AttestationHeader;
    pub fn public_key(&self) -> ed25519_dalek::VerifyingKey;
    pub fn attestation_json(&self) -> serde_json::Value;
}
```

#### With `#[mrsigner]`

```rust
impl TeeState {
    pub fn signer(&self) -> &SignerState;
    pub fn signer_mut(&mut self) -> &mut SignerState;
    pub fn check_rotation(&mut self) -> Result<bool, SdkError>;
    pub fn rotate_master_key(&mut self) -> Result<(), SdkError>;
}
```

#### For each `#[external]` type

```rust
impl TeeState {
    pub fn encrypt_type4(&self, value: &Type4) -> Result<EncryptedType4, SdkError>;
    pub fn decrypt_type4(&self, encrypted: &EncryptedType4) -> Result<Type4, SdkError>;
}
```

## Field naming convention

The macro converts type names to snake_case for field names:

| Type name | Field name | Accessor |
|-----------|-----------|----------|
| `PriceCache` | `price_cache` | `.price_cache()` |
| `BtcWallet` | `btc_wallet` | `.btc_wallet()` |
| `OracleConfig` | `oracle_config` | `.oracle_config()` |

## Complete example

```rust
use guarantee::{attest, state, Encrypted, crypto::Encryptable};
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Default, Clone, Debug)]
struct SessionData {
    request_count: u64,
}

#[derive(Serialize, Deserialize, Default, Clone, Debug)]
struct WalletKeys {
    private_key_hex: String,
}

#[derive(Encrypted, Serialize, Deserialize, Clone, Debug)]
struct AuditLog {
    event: String,
    #[encrypt]
    details: String,
    timestamp: String,
}

state! {
    #[mrenclave]
    SessionData,

    #[mrsigner]
    WalletKeys,

    #[external]
    AuditLog,
}

// Now you have:
// - TeeState::initialize(path)
// - state.enclave().session_data()
// - state.signer().wallet_keys()
// - state.encrypt_audit_log(&entry)
// - state.decrypt_audit_log(&encrypted)
// - state.sign_response(body, req_id)
// - state.attestation_json()
```
