# Attestation

Attestation is the process of proving that code is running inside a genuine Trusted Execution Environment. Guarantee automates this for every HTTP response.

## Core types

### `MrEnclave`

A 32-byte SHA-256 hash of the enclave's binary contents. If you change a single byte of code and redeploy, the MRENCLAVE changes. This is the enclave's *identity*.

```rust
use guarantee::MrEnclave;

let measurement = MrEnclave::new([0u8; 32]);
println!("Enclave measurement: {}", measurement); // hex string
```

### `MrSigner`

A 32-byte hash of the key used to sign the enclave. All enclaves signed by the same key share an MRSIGNER. This is useful for key continuity across deployments — MRSIGNER-sealed data persists even when the binary changes.

```rust
use guarantee::MrSigner;

let signer = MrSigner::new([0u8; 32]);
```

### `StartupQuote`

The SGX quote produced at enclave startup. It cryptographically binds the enclave's identity (MRENCLAVE + MRSIGNER) to the ephemeral Ed25519 public key.

```rust
pub struct StartupQuote {
    pub raw_quote: Vec<u8>,                         // Raw DCAP quote bytes
    pub mr_enclave: MrEnclave,                      // Enclave measurement
    pub mr_signer: MrSigner,                        // Signer identity
    pub attested_public_key: ed25519_dalek::VerifyingKey,  // Bound key
    pub produced_at: DateTime<Utc>,                 // Timestamp
}
```

### `AttestationHeader`

The per-response signature attached to every HTTP response from an `#[attest]` handler.

```rust
pub struct AttestationHeader {
    pub version: u32,             // Always 1
    pub signature_b64: String,    // Base64 Ed25519 signature
    pub payload_hash_hex: String, // SHA-256(body ‖ ts ‖ request_id)
    pub timestamp_ms: u64,        // Unix timestamp in milliseconds
    pub public_key_hex: String,   // Hex Ed25519 public key
}
```

The header serializes to:

```
v=1; sig=<base64>; hash=<hex>; ts=<ms>; key=<hex>
```

And can be parsed back:

```rust
use guarantee::AttestationHeader;

let header = AttestationHeader::from_header_value(
    "v=1; sig=abc123; hash=def456; ts=1711234567890; key=789abc"
)?;
```

## Gramine interface

In enclave mode, Guarantee communicates with the SGX hardware via Gramine's `/dev/attestation` pseudo-filesystem:

```rust
use guarantee::gramine;

// Write the public key hash as user report data
let mut report_data = [0u8; 64];
report_data[..32].copy_from_slice(&sha256_of_pubkey);
gramine::write_user_report_data(&report_data)?;

// Read the DCAP quote
let quote_bytes = gramine::read_quote()?;
```

:::warning
These functions only work inside a Gramine SGX enclave. In dev mode, `EnclaveAttestor::initialize()` handles this automatically with mock values.
:::

## The `.well-known` endpoint

Every Guarantee service should expose the startup attestation at:

```
GET /.well-known/tee-attestation
```

This returns a JSON document containing the startup quote, public key, and attestation mode. Callers fetch this once and use it to verify all subsequent response signatures.

```json
{
  "version": 1,
  "public_key": "3b6a27bcceb6a42d...",
  "mode": "every-response",
  "quote": {
    "raw_quote_b64": "AgAAAEoLAAAL...",
    "mr_enclave": "a1b2c3d4e5f6...",
    "mr_signer": "f6e5d4c3b2a1...",
    "produced_at": "2024-03-23T12:00:00Z"
  }
}
```
