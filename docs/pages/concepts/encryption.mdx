# Field-Level Encryption

Guarantee provides automatic field-level encryption for storing sensitive data in external systems (databases, caches, message queues). The encryption key never leaves the enclave.

## How it works

1. The `#[derive(Encrypted)]` macro generates an encrypted variant of your struct
2. Fields marked with `#[encrypt]` are encrypted with AES-256-GCM
3. Non-encrypted fields pass through unchanged
4. The encryption key is derived from the MRSIGNER master key via HKDF-SHA256

## Defining encrypted types

```rust
use guarantee::{Encrypted, crypto::Encryptable};
use serde::{Serialize, Deserialize};

#[derive(Encrypted, Serialize, Deserialize, Clone, Debug)]
struct UserRecord {
    user_id: String,         // Plaintext — queryable in DB
    #[encrypt]
    ssn: String,             // Encrypted at rest
    #[encrypt]
    bank_account: String,    // Encrypted at rest
    email: String,           // Plaintext
}
```

This generates:

```rust
// Auto-generated by #[derive(Encrypted)]
struct EncryptedUserRecord {
    user_id: String,
    ssn: String,           // Contains "enc:v1:k1:<nonce>:<ciphertext>"
    bank_account: String,  // Contains "enc:v1:k1:<nonce>:<ciphertext>"
    email: String,
}
```

## Encrypting and decrypting

With the `state!` macro's `#[external]` tier, encryption methods are generated on `TeeState`:

```rust
state! {
    #[mrsigner]
    MasterConfig,

    #[external]
    UserRecord,
}

// In your handler:
let state = state.read().await;

// Encrypt before storing
let encrypted: EncryptedUserRecord = state.encrypt_user_record(&user)?;
db.insert(encrypted).await?;

// Decrypt after loading
let encrypted = db.find_by_id(id).await?;
let user: UserRecord = state.decrypt_user_record(&encrypted)?;
```

## Manual encryption

You can also use the low-level crypto functions directly:

```rust
use guarantee::crypto::{encrypt_field, decrypt_field};

let key: [u8; 32] = /* your key */;

let ciphertext = encrypt_field("sensitive data", &key)?;
// "enc:v1:<nonce_hex>:<ciphertext_hex>"

let plaintext = decrypt_field(&ciphertext, &key)?;
// "sensitive data"
```

### Versioned encryption

For key rotation support:

```rust
use guarantee::crypto::{encrypt_field_versioned, decrypt_field_versioned, derive_key};

let master_key: [u8; 32] = /* from MRSIGNER state */;
let derived = derive_key(&master_key, b"user-records");

let ciphertext = encrypt_field_versioned("secret", &derived, 1, b"user-records")?;
// "enc:v1:k1:<nonce_hex>:<ciphertext_hex>"

// Decryption automatically tries retired keys
let plaintext = decrypt_field_versioned(
    &ciphertext,
    &current_key,
    current_version,
    &retired_keys,
    b"user-records",
)?;
```

## Ciphertext format

Encrypted fields use a self-describing format:

```
enc:v1:<nonce_hex>:<ciphertext_hex>           // Unversioned
enc:v1:k<version>:<nonce_hex>:<ciphertext_hex> // Versioned (with key rotation)
```

- `enc` — magic prefix for detection
- `v1` — format version
- `k<N>` — key version (for rotation)
- `<nonce>` — 12-byte AES-GCM nonce (hex-encoded)
- `<ciphertext>` — AES-256-GCM ciphertext with 16-byte auth tag (hex-encoded)

## Key derivation

Encryption keys are derived from the MRSIGNER master key using HKDF-SHA256 with a purpose-specific context:

```rust
use guarantee::crypto::derive_key;

let master_key: [u8; 32] = /* from signer state */;
let user_key = derive_key(&master_key, b"user-records");
let audit_key = derive_key(&master_key, b"audit-log");
```

Different purposes produce different keys, so compromising one derivation doesn't affect others.

## Key rotation

The MRSIGNER state includes automatic key rotation (default: every 90 days). When a key rotates:

1. The old key is added to the `retired_keys` list
2. New encryptions use the new key version
3. Decryptions try the current key first, then fall back to retired keys
4. Retired keys can have expiry dates

```rust
pub struct RetiredKeyEntry {
    pub version: u32,
    pub key: [u8; 32],
    pub retired_at: String,           // RFC3339
    pub expires_at: Option<String>,   // RFC3339
}
```

:::info
Only `String` fields can be marked with `#[encrypt]`. If you need to encrypt other types, serialize them to a String first.
:::
