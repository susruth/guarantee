# WebSocket Over RA-TLS

Real-time attested data streaming. The TLS handshake verifies the SGX attestation quote — by the time the WebSocket connection is established, the client has cryptographic proof they're talking to a genuine enclave. No per-message signing overhead.

## Run it

```bash
# HTTP mode (dev, no SGX)
cargo run --example websocket-ratls

# With RA-TLS (attested TLS)
cargo run --example websocket-ratls --features ra-tls
```

## Connect

# Install wscat if needed: npm install -g wscat

## How It Works

The example combines three patterns:

1. **WebSocket over RA-TLS** — The connection runs over TLS where the certificate carries the SGX attestation quote. Verified once at handshake, all subsequent messages are authenticated to the enclave.

2. **`state!` for stream management** — `StreamStats` tracks connections and messages (MRENCLAVE-sealed, resets per deploy). `StreamConfig` holds settings (MRSIGNER-sealed, persists).

3. **`#[attest]` for REST endpoints** — The `/stats` endpoint returns attested JSON alongside the WebSocket stream.

## Architecture

```
Client                          Enclave
  │                                │
  ├── TLS Handshake ──────────────►│  ← SGX quote verified here
  │   (RA-TLS: cert has quote)     │
  │◄── Connection Established ─────┤
  │                                │
  ├── {"type":"subscribe"} ───────►│  ← All messages encrypted
  │◄── {"type":"subscribed"} ──────┤    in the TLS tunnel
  │                                │
  │◄── {"type":"price_update"} ────┤  ← Background publisher
  │◄── {"type":"price_update"} ────┤    pushes via broadcast channel
  │                                │
```

## Full Source

```rust
use axum::{
    extract::ws::{Message, WebSocket, WebSocketUpgrade},
    extract::Extension,
    response::{IntoResponse, Json},
    routing::get,
    Router,
};
use guarantee::{attest, state};
use serde::{Deserialize, Serialize};
use std::sync::Arc;
use tokio::sync::{broadcast, RwLock};

// Sealed state for stream management
#[derive(Serialize, Deserialize, Default, Clone, Debug)]
struct StreamStats {
    total_connections: u64,
    total_messages_sent: u64,
    active_subscriptions: u64,
}

#[derive(Serialize, Deserialize, Default, Clone, Debug)]
struct StreamConfig {
    max_clients: u32,
    price_update_interval_ms: u64,
}

state! {
    #[mrenclave]
    StreamStats,

    #[mrsigner]
    StreamConfig,
}

// WebSocket upgrade handler
async fn ws_handler(
    ws: WebSocketUpgrade,
    Extension(tee_state): Extension<Arc<RwLock<TeeState>>>,
    Extension(price_tx): Extension<broadcast::Sender<PriceUpdate>>,
) -> impl IntoResponse {
    ws.on_upgrade(move |socket| handle_socket(socket, tee_state, price_tx))
}

async fn handle_socket(
    mut socket: WebSocket,
    tee_state: Arc<RwLock<TeeState>>,
    price_tx: broadcast::Sender<PriceUpdate>,
) {
    // Track connection
    {
        let mut state = tee_state.write().await;
        state.enclave_mut().stream_stats.total_connections += 1;
    }

    let mut price_rx = price_tx.subscribe();
    let mut subscribed_to_prices = false;

    loop {
        tokio::select! {
            msg = socket.recv() => {
                match msg {
                    Some(Ok(Message::Text(text))) => {
                        // Handle subscribe/unsubscribe/ping
                        if text.contains("subscribe") {
                            subscribed_to_prices = true;
                        }
                    }
                    _ => break,
                }
            }
            update = price_rx.recv() => {
                if let Ok(price) = update {
                    if subscribed_to_prices {
                        let json = serde_json::to_string(&price).unwrap_or_default();
                        if socket.send(Message::Text(json.into())).await.is_err() {
                            break;
                        }
                    }
                }
            }
        }
    }
}

// REST endpoint — attested per-response
#[attest]
async fn get_stats() -> Json<serde_json::Value> {
    let state = tee_state.read().await;
    let stats = state.enclave().stream_stats();
    Json(serde_json::json!({
        "total_connections": stats.total_connections,
        "total_messages_sent": stats.total_messages_sent,
    }))
}

#[tokio::main]
async fn main() {
    let tee_state = TeeState::initialize(std::path::Path::new("./sealed"))
        .expect("Failed to initialize TeeState");
    let tee_state = Arc::new(RwLock::new(tee_state));

    let (price_tx, _) = broadcast::channel::<PriceUpdate>(256);
    tokio::spawn(publish_prices(price_tx.clone()));

    let app = Router::new()
        .route("/ws", get(ws_handler))
        .route("/stats", get(get_stats))
        .layer(Extension(tee_state))
        .layer(Extension(price_tx));

    // Dev mode: plain HTTP
    let listener = tokio::net::TcpListener::bind("0.0.0.0:8080").await.unwrap();
    axum::serve(listener, app).await.unwrap();

    // With RA-TLS:
    // guarantee::serve_ra_tls(app, "ws-service", "0.0.0.0:8443", "0.0.0.0:8080").await?;
}
```

## Key Points

- **No per-message signing** — RA-TLS authenticates the entire channel at the TLS layer. Every message is encrypted and authenticated to the attested enclave.
- **`broadcast::channel`** — Tokio's broadcast channel fans out price updates to all connected WebSocket clients.
- **`#[attest]` on REST** — The `/stats` endpoint uses per-response attestation (belt and suspenders alongside RA-TLS).
- **Dev mode works** — Without `--features ra-tls`, runs as plain HTTP WebSocket. Same code, same protocol, no SGX needed.

## Client Messages

| Type | Payload | Description |
|------|---------|-------------|
| `subscribe` | `{"type":"subscribe","channel":"prices"}` | Start receiving price updates |
| `unsubscribe` | `{"type":"unsubscribe","channel":"prices"}` | Stop receiving updates |
| `ping` | `{"type":"ping"}` | Heartbeat — server replies with `pong` |

## Server Messages

| Type | Description |
|------|-------------|
| `connected` | Sent on connection — includes welcome message |
| `subscribed` | Confirms subscription to a channel |
| `unsubscribed` | Confirms unsubscription |
| `price_update` | Real-time price data (symbol, price, timestamp) |
| `pong` | Reply to client ping |
| `error` | Invalid message format |
