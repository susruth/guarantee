# Encrypted Postgres

Store sensitive user data in PostgreSQL with automatic field-level encryption. The database only ever sees ciphertext for `#[encrypt]` fields — only the enclave can decrypt.

## What this demonstrates

- `#[derive(Encrypted)]` with `#[encrypt]` field annotations
- Encrypt before storing, decrypt after loading
- Raw ciphertext inspection (`/users/:id/raw`)
- The database never sees plaintext sensitive data

## Full source

```rust
use axum::{
    extract::{Extension, Path},
    http::StatusCode,
    response::{IntoResponse, Json},
    routing::{get, post},
    Router,
};
use guarantee::{state, attest, Encrypted, crypto::Encryptable};
use serde::{Deserialize, Serialize};
use std::sync::Arc;
use tokio::sync::RwLock;

#[derive(Encrypted, Serialize, Deserialize, Clone, Debug)]
struct UserRecord {
    user_id: String,
    #[encrypt]
    ssn: String,       // Encrypted at rest
    email: String,     // Plaintext (queryable)
}

#[derive(Serialize, Deserialize, Default, Clone, Debug)]
struct DbConfig {
    connection_url: String,
}

state! {
    #[mrenclave]
    DbConfig,

    #[mrsigner]
    DbConfig,

    #[external]
    UserRecord,
}

// In-memory mock DB (replace with sqlx::PgPool in production)
struct MockDb {
    records: tokio::sync::RwLock<
        std::collections::HashMap<String, EncryptedUserRecord>
    >,
}

#[attest]
async fn create_user(
    Extension(state): Extension<Arc<RwLock<TeeState>>>,
    Extension(db): Extension<Arc<MockDb>>,
    Json(user): Json<UserRecord>,
) -> impl IntoResponse {
    let state = state.read().await;

    // Encrypt sensitive fields before storing
    let encrypted = match state.encrypt_user_record(&user) {
        Ok(e) => e,
        Err(e) => {
            return (
                StatusCode::INTERNAL_SERVER_ERROR,
                Json(serde_json::json!({"error": e.to_string()})),
            ).into_response()
        }
    };

    // What gets stored in the DB:
    // user_id: "alice"                           ← plaintext (queryable)
    // ssn:     "enc:v1:a1b2c3d4...:7f8a9b..."   ← AES-256-GCM ciphertext
    // email:   "alice@example.com"               ← plaintext

    db.records.write().await
        .insert(encrypted.user_id.clone(), encrypted);

    (
        StatusCode::CREATED,
        Json(serde_json::json!({"status": "created", "user_id": user.user_id})),
    ).into_response()
}

#[attest]
async fn get_user(
    Extension(state): Extension<Arc<RwLock<TeeState>>>,
    Extension(db): Extension<Arc<MockDb>>,
    Path(user_id): Path<String>,
) -> impl IntoResponse {
    let state = state.read().await;
    let records = db.records.read().await;

    let encrypted = match records.get(&user_id) {
        Some(r) => r,
        None => return (
            StatusCode::NOT_FOUND,
            Json(serde_json::json!({"error": "User not found"})),
        ).into_response(),
    };

    // Decrypt inside the enclave — only we have the key
    match state.decrypt_user_record(encrypted) {
        Ok(user) => Json(serde_json::json!({
            "user_id": user.user_id,
            "ssn": user.ssn,
            "email": user.email,
        })).into_response(),
        Err(e) => (
            StatusCode::INTERNAL_SERVER_ERROR,
            Json(serde_json::json!({"error": format!("Decryption failed: {e}")})),
        ).into_response(),
    }
}

// Show what the database actually stores (ciphertext)
async fn get_user_raw(
    Extension(db): Extension<Arc<MockDb>>,
    Path(user_id): Path<String>,
) -> impl IntoResponse {
    let records = db.records.read().await;
    match records.get(&user_id) {
        Some(encrypted) => Json(serde_json::json!({
            "user_id": encrypted.user_id,
            "ssn": encrypted.ssn,
            "email": encrypted.email,
            "_note": "The ssn field is AES-256-GCM encrypted."
        })).into_response(),
        None => (
            StatusCode::NOT_FOUND,
            Json(serde_json::json!({"error": "User not found"})),
        ).into_response(),
    }
}
```

## Run it

```bash
cargo run --example postgres-encrypted
```

## Try it

```bash
# Create a user with sensitive data
curl -X POST http://localhost:8080/users \
  -H 'Content-Type: application/json' \
  -d '{"user_id":"alice","ssn":"123-45-6789","email":"alice@example.com"}'
```

```json
{"status": "created", "user_id": "alice"}
```

```bash
# Retrieve the user (decrypted in the enclave)
curl http://localhost:8080/users/alice | jq
```

```json
{
  "user_id": "alice",
  "ssn": "123-45-6789",
  "email": "alice@example.com"
}
```

```bash
# See what the database actually stores
curl http://localhost:8080/users/alice/raw | jq
```

```json
{
  "user_id": "alice",
  "ssn": "enc:v1:k1:a1b2c3d4e5f6a1b2c3d4e5f6:7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3d4e5f6a7b8c9d0e1f2a3b4",
  "email": "alice@example.com",
  "_note": "The ssn field is AES-256-GCM encrypted."
}
```

The SSN is stored as ciphertext. Even with full database access, an attacker cannot read it — only the enclave's master key can decrypt.

## Key patterns

### Encrypt on write, decrypt on read

```rust
// Before storing → encrypt
let encrypted = state.encrypt_user_record(&user)?;
db.insert(encrypted).await;

// After loading → decrypt
let encrypted = db.find_by_id(id).await?;
let user = state.decrypt_user_record(&encrypted)?;
```

### Selective encryption

Only fields marked `#[encrypt]` are encrypted. Others pass through as plaintext, keeping them queryable by the database:

```rust
#[derive(Encrypted, Serialize, Deserialize, Clone, Debug)]
struct UserRecord {
    user_id: String,    // Plaintext — can use as primary key
    #[encrypt]
    ssn: String,        // Encrypted — database sees ciphertext
    email: String,      // Plaintext — can query by email
}
```

### Production usage with sqlx

Replace the mock DB with real PostgreSQL:

```rust
let pool = sqlx::PgPool::connect(&database_url).await?;

// Insert encrypted record
sqlx::query("INSERT INTO users (user_id, ssn, email) VALUES ($1, $2, $3)")
    .bind(&encrypted.user_id)
    .bind(&encrypted.ssn)    // Ciphertext stored in DB
    .bind(&encrypted.email)
    .execute(&pool)
    .await?;
```
